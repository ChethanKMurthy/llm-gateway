the problem

Sending every request to a frontier model is the
most expensive way to be approximately right.

A translation, an FAQ lookup, and a hard reasoning task do not need the same model — but most stacks send all three to the biggest one, pay the top rate, and re-answer questions they already answered an hour ago.

  • Duplicate & near-duplicate prompts re-billed in full.
  • Cheap tasks charged at frontier prices.
  • One provider down takes the whole feature with it.
  • No idea what any of it costs until the invoice.
how it works

routing intelligence

It learns which model is actually best — per kind of work.

The router is a Thompson-sampling contextual bandit. Each intent is a context; each candidate model is an arm; the reward blends quality, cost and latency. It starts from informed priors and sharpens toward whatever performs on your traffic.
